- 启用取消功能
- GetCancellationTokenOnDestroy
- _场景_：你发起加载一张图片，还没加载完，玩家就把 UI 关掉了。如果没有取消机制，任务完成后会试图把图片赋值给一个已经销毁的 Image 组件，导致 `MissingReferenceException` 或内存泄漏。
- 可以接一个progress的回调

``` csharp
    // .WithCancellation 会启用取消功能，GetCancellationTokenOnDestroy 表示获取一个依赖对象生命周期的 Cancel 句柄，当对象被销毁时，将会调用这个 Cancel 句柄，从而实现取消的功能
    // 在 Unity 2022.2之后，您可以在 MonoBehaviour 中使用`destroyCancellationToken`
    var asset2 = await Resources.LoadAsync<TextAsset>("bar").WithCancellation(this.GetCancellationTokenOnDestroy());

    // .ToUniTask 可接收一个 progress 回调以及一些配置参数，Progress.Create 是 IProgress<T> 的轻量级替代方案
    var asset3 = await Resources.LoadAsync<TextAsset>("baz").ToUniTask(Progress.Create<float>(x => Debug.Log(x)));
```

- WaitUntil及部分
==
### 1. `UniTask.WaitUntil(() => isActive == false)`

这是最传统的“条件等待”。

- **逻辑**：每一帧都会运行括号里的 Lambda 表达式。**只要表达式返回 `true`，等待就结束**。
    
- **关注点**：它只关心**结果**是否满足条件。
    
- **场景**：你明确知道你想要什么状态。
    
    - _例子_：我不管 `isActive` 变了多少次，我只要它变成 `false` 的那一刻，我就继续执行。
        
- **返回值**：`void`（没有返回值，只是通关放行）。
==

### 2. `UniTask.WaitUntilValueChanged(this, x => x.isActive)`

这是监控“变化”的方法。

- **逻辑**：它会记录 `isActive` 的**当前值**。每一帧它都会去检查这个值，一旦发现**当前值 != 上一帧的值**，等待就结束。
    
- **关注点**：它关心的是**改变**这个动作。
    
- **场景**：你不确定未来会变成什么，你只是想监听“状态切换”。
    
    - 如果 `isActive` 现在是 `true`，它等到变成 `false` 时结束。
        
    - 如果 `isActive` 现在是 `false`，它等到变成 `true` 时结束。
        
- **返回值**：`T`（返回变化后的那个新值）。


``` csharp
    // yield return WaitUntil 的替代方案
    await UniTask.WaitUntil(() => isActive == false);

    // WaitUntil 扩展，指定某个值改变时触发
    await UniTask.WaitUntilValueChanged(this, x => x.isActive);
```

### 1. 场景：你要做两件事 —— “切菜” 和 “烧水”

#### 方案 A：异步/协程 (Async/Coroutine) —— **一个人干活**

- **核心**：**并发 (Concurrency)**。你只有**一只手**（一个主线程）。
    
- **操作**：
    
    1. 你把水壶放在炉子上（发起 `await`）。
        
    2. 利用烧水的时间，你转身去切菜（Unity 渲染画面）。
        
    3. 听到水开了（Task 完成），你放下菜刀，去处理开水。
        
- **结果**：菜切好了，水也烧开了。
    
- **特点**：虽然看起来两件事都在推进，但**任何时刻只有一个动作在发生**。
    
- **瓶颈**：如果你切菜切得太累（主线程计算太重），你就没空去看水开了没有；或者如果你一直盯着水壶（死循环），菜就没人切了。
    

#### 方案 B：多线程 (Multithreading) —— **两个人干活**

- **核心**：**并行 (Parallelism)**。你有**两只手**，甚至叫了一个**帮手**（后台线程）。
    
- **操作**：
    
    1. 你叫帮手去烧水（`Task.Run`）。
        
    2. 你自己专心切菜（Unity 渲染画面）。
        
    3. 你们两个**同时**在干活，互不干扰。
        
- **结果**：菜切好了，水也烧开了。
    
- **特点**：**真正的同时进行**。
    
- **瓶颈**：你需要给帮手发工资（线程切换开销），而且帮手不能碰你的菜刀（**线程安全问题**，后台线程不能碰 Unity 组件）。

``` csharp
    // 多线程示例，在此行代码后的内容都运行在一个线程池上
    await UniTask.SwitchToThreadPool();

    /* 工作在线程池上的代码 */

    // 转回主线程（等同于 UniRx 的`ObserveOnMainThread`）
    await UniTask.SwitchToMainThread();
```

这段代码包含了两部分信息：一句复杂的 **C# 语法糖（一行代码完成多步判断）**，以及注释里关于 **UniTask 返回类型** 的说明。

我来把它们拆解开解释。

---

### 1. 代码逻辑详解：那一行 `return` 到底干了什么？

C#

```
return (asset as TextAsset)?.text ?? throw new InvalidOperationException("Asset not found");
```

这行代码极其精简，它在一行里完成了 **“类型转换 + 空检查 + 取值 + 异常抛出”** 4 件事。

我们可以把它“翻译”成普通的、啰嗦的 C# 代码，你就明白了：

C#

```
// 1. 尝试把 asset 转换成 TextAsset 类型
TextAsset textAsset = asset as TextAsset;

// 2. 检查转换是否成功（是不是 null）
if (textAsset != null)
{
    // 3. 如果成功，返回它的文本内容
    return textAsset.text;
}
else
{
    // 4. 如果 asset 是 null，或者它不是 TextAsset（比如是个图片），直接报错！
    throw new InvalidOperationException("Asset not found");
}
```

#### 语法糖拆解：

1. **`as` (安全转换)**：尝试把 `asset` 转成 `TextAsset`。如果失败（比如 `asset` 其实是一张图片，或者 `asset` 本身就是空），它不会报错，而是返回 `null`。
    
2. **`?.` (空值传播)**：如果前面的 `(asset as TextAsset)` 是 `null`，那么后面的 `.text` 就不会执行，整个表达式直接返回 `null`。
    
3. **`??` (空合并运算符)**：这是核心！它的意思是 **“如果是左边是 null，就执行右边”**。
    
    - 如果左边拿到了文本 -> 返回文本。
        
    - 如果左边是 `null` -> 执行 `throw ...`。
        

---

### 2. 注释含义详解：UniTask 的三种返回类型

注释里提到了 `UniTask<T>`, `UniTask`, `UniTaskVoid`，这是你在写异步方法时必须做的**三选一**决定。

#### A. `UniTask<T>` (带返回值的异步)

- **含义**：我不光要等你做完，我还要你**给我一个结果**。
    
- **场景**：加载资源、计算数值、读取文件。
    
- **例子**：
    
    C#
    
    ```
    // 就像你的代码：返回一个 string
    async UniTask<string> GetTextAsync() { ... return "Hello"; }
    
    // 调用时：
    string s = await GetTextAsync(); // 拿到 "Hello"
    ```
    

#### B. `UniTask` (不带返回值的异步)

- **含义**：我等你做完就行，**不需要结果**（相当于 `void`，但可以被 `await`）。
    
- **场景**：播放一个动画、等待 3 秒、执行一段流程。
    
- **例子**：
    
    C#
    
    ```
    async UniTask PlayEffectAsync() { ... }
    
    // 调用时：
    await PlayEffectAsync(); // 等动画播完，然后继续
    ```
    

#### C. `UniTaskVoid` (不可等待的异步 / "Fire and Forget")

- **含义**：**射后不理**。我触发这个任务就不管了，我**没法等待**它，也不关心它什么时候结束。
    
- **场景**：**Unity 的事件入口**（如 `Start`、`Button.OnClick`）。这是唯一推荐使用 `UniTaskVoid` 的地方。
    
- **例子**：
    
    C#
    
    ```
    // ✅ 正确：作为入口，启动异步逻辑
    async UniTaskVoid Start() 
    {
        await LoadGameAsync();
    }
    
    // ❌ 错误：你不能 await 一个 UniTaskVoid！
    // await Start(); // 编译报
    ```
